# Sustainability Assessment of a Small Language Model (SLM)

Codebase for the DTU course case study: training a small GPT-style character model and measuring sustainability impacts from both training and inference.

## Repository Structure

```text
.
├── data/
│   ├── prepare.py
│   ├── train.bin / val.bin / meta.pkl   # generated by prepare.py
│   └── input.txt                         # downloaded Tiny Shakespeare text
├── src/
│   ├── model.py
│   ├── train.py
│   ├── prompt.py
│   ├── run_experiments.py
│   ├── run_inference_experiments.py
│   └── plot_results.py
├── out/                                  # generated training/inference outputs
├── env_requirements.yaml
└── README.md
```

## Setup

```bash
conda env create -f env_requirements.yaml
conda activate slm-sustainability
```

## Quick Start

Run from repository root:

```bash
python data/prepare.py
python src/run_experiments.py
python src/run_inference_experiments.py --ckpt_path out/ckpt_baseline.pt
python src/plot_results.py
```

## Scripts

### 1) Prepare data

```bash
python data/prepare.py
```

What it does:
- Downloads Tiny Shakespeare to `data/input.txt` (if missing)
- Creates char-level vocabulary
- Writes `data/train.bin`, `data/val.bin`, `data/meta.pkl`

### 2) Train one scenario

```bash
python src/train.py \
  --scenario_name baseline \
  --max_iters 2000 \
  --n_layer 4 \
  --n_head 4 \
  --batch_size 16
```

Arguments:
- `--scenario_name` (default: `baseline`)
- `--max_iters` (default: `2000`)
- `--n_layer` (default: `4`)
- `--n_head` (default: `4`)
- `--batch_size` (default: `16`)

Outputs in `out/`:
- `<scenario_name>_loss.txt`
- `ckpt.pt` (latest generic checkpoint)
- `ckpt_<scenario_name>.pt` (scenario-specific checkpoint)
- `emissions.csv` (CodeCarbon log, appended over runs)

### 3) Run all training scenarios (Task 2, scenarios 1-3 + baseline)

```bash
python src/run_experiments.py
```

Runs:
- `baseline`
- `scenario_1_short`
- `scenario_1_long`
- `scenario_2_scaled_arch`
- `scenario_3_batch_8`
- `scenario_3_batch_32`

### 4) Generate text from a checkpoint

```bash
python src/prompt.py \
  --ckpt_path out/ckpt_baseline.pt \
  --prompt "To be, or not to be" \
  --max_new_tokens 200 \
  --temperature 1.0 \
  --top_k 50
```

Arguments:
- `--ckpt_path` (default: `out/ckpt.pt`)
- `--prompt`
- `--max_new_tokens`
- `--temperature`
- `--top_k`

### 5) Run inference scenarios (Task 2, scenarios 4 and 5)

```bash
python src/run_inference_experiments.py \
  --ckpt_path out/ckpt_baseline.pt \
  --results_csv out/inference_scenarios_results.csv \
  --repeat 3 \
  --temperature 1.0 \
  --top_k 50
```

Scenario logic:
- Scenario 4: vary prompt length (20, 80, 200, 400 chars), fixed output length
- Scenario 5: vary output length (`max_new_tokens`: 50, 200, 500), fixed prompt

Output:
- `out/inference_scenarios_results.csv`

### 6) Plot Task 2 training + inference results

```bash
python src/plot_results.py \
  --out_dir out \
  --emissions_csv out/emissions.csv \
  --inference_csv out/inference_scenarios_results.csv \
  --output_img task2_results.png \
  --output_csv out/task2_summary.csv \
  --output_inference_csv out/task2_inference_summary.csv
```

Defaults:
- `--out_dir out`
- `--emissions_csv out/emissions.csv`
- `--inference_csv out/inference_scenarios_results.csv`
- `--output_img task2_results.png`
- `--output_csv out/task2_summary.csv`
- `--output_inference_csv out/task2_inference_summary.csv`

Expected input files:
- `out/emissions.csv` with at least `timestamp`, `project_name`, `emissions`
- `out/<scenario>_loss.txt` for all Task 2 training scenarios listed above
- `out/inference_scenarios_results.csv` from `run_inference_experiments.py`

Outputs:
- `task2_results.png`
- `out/task2_summary.csv`
- `out/task2_inference_summary.csv`

## Notes

- Device is auto-selected in scripts: `cuda` -> `mps` -> `cpu`.
- `src/model.py` contains the decoder-only GPT architecture used by both training and inference.
- `plot_results.py` combines training (S1-3) and inference (S4-5) in one figure.
- Inference panel in the figure shows CO2e only (mean mg CO2e) for S4 and S5.
- `out/task2_inference_summary.csv` still includes both emissions and elapsed-time aggregates.
